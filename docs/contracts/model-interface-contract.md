# Model Interface Contract

Status: draft

Owner modules:

- `MOD-MODEL-CODEGEN`
- `MOD-ALG-EVAL`
- `MOD-CORE-SEMANTICS`

Related modules:

- `MOD-SOLVER-ROUTE`
- `MOD-DEBUG-SNAPSHOT`

Coverage matrix:

- [`../testing/contract-coverage-matrix.md`](../testing/contract-coverage-matrix.md)

## Scope

Static C++ model interface consumed by MiniSolver, whether handwritten or
generated by MiniModel.

## Contract IDs

| ID | Requirement | Evidence status |
| --- | --- | --- |
| `MODEL-001` | Model must provide compile-time `NX`, `NU`, `NC`, and `NP`. | `covered` |
| `MODEL-002` | Model must provide state, control, and parameter name arrays matching dimensions. | `covered` |
| `MODEL-003` | Model must provide the required stage cost, dynamics, and constraint packet functions for its configured use. | `covered` |
| `MODEL-004` | Optional true, terminal, SOC, generated-integrator, and fingerprint hooks are detected by traits. | `covered` |
| `MODEL-005` | Generated integrator metadata must match the active solver integrator before fused/generated paths are used. | `covered` |
| `MODEL-006` | Model fingerprint participates in snapshot/replay compatibility checks when available. | `covered` |

## Inputs And Outputs

| Kind | Values |
| --- | --- |
| Inputs | `KnotPoint`, `SolverConfig`, `dt`, terminal flag |
| Outputs | Cost, dynamics, constraint, soft weight, and metadata packets |

## Failure Semantics

| Failure | Required behavior |
| --- | --- |
| Missing required static model member | Compile failure. |
| Optional hook missing | Trait-defined fallback. |
| Model mismatch on snapshot load | `SnapshotStatus::ModelMismatch` when policy rejects mismatch. |
