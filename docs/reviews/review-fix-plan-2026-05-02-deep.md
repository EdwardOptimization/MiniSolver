## Open Follow-Up Order

Current recommended sequence after the May 3 status re-anchor:

1. **N-OBS-1**: continue structured diagnostics only when the owning contract is
   ready. Implemented counters now cover degraded Riccati freeze stages,
   regularization escalations, SOC attempts/accepts/rejects, and restoration
   attempt/success counts. Scaling status waits for N-MOD-2; factorization
   status waits for linear-solver result expansion.
2. **N-API-1 + N-OBS-2**: continue the accepted
   [`api-logger-boundary-design.md`](../architecture/api-logger-boundary-design.md)
   in red-test batches. API setter `ApiStatus` returns and checked scalar
   getters are in place. The logger now routes through a central
   callback-capable backend; remaining logger work is embedded fixed-buffer/no-
   stream behavior and release documentation. Snapshot remains out of scope.
3. **N-MOD-2**: continue scaling from
   [`scaling-normalization-design.md`](../architecture/scaling-normalization-design.md).
   Stage 0 through the conservative `RUIZ_EQUILIBRATION` profile are done. Next
   steps are coordinate-level variable scaling only if benchmark evidence
   justifies the extra complexity.
4. **N-CONV-5** external benchmark follow-up belongs to MiniSolver-Bench. Reopen
   solver-convergence work here only when a generic MiniSolver bug is reproduced
   by a local test, reduced reproducer, or benchmark-independent model contract.
5. **N-THEORY-2** Pareto-frontier filter history, after N-THEORY-1 semantics
   are stable.
6. **N-EMBED-1** embedded profile: logger/exception/container boundaries, ARM
   cross-compile, and binary-size measurement. This is a release-readiness item,
   not a solver-algorithm blocker.
7. **N-THEORY-4** inertia detection: defer until benchmark evidence or real
   solver failures show regularization escalation is insufficient.
